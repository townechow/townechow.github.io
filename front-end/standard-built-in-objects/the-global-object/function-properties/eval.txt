c:T7cb,<h1>eval</h1>
<p><code>eval()</code> 函数用于计算并执行以字符串表示的 JavaScript 代码。<code>eval()</code> 函数使 JavaScript 可以动态执行 JavaScript 源代码。</p>
<p><code>eval()</code> 函数属于 <code>Global</code> 对象，所有主流浏览器均支持该函数。</p>
<h2>语法</h2>
<pre class="hljs"><code class="language-js"><span class="hljs-built_in">eval</span>(code);
</code></pre>
<table>
<thead>
<tr>
<th>参数</th>
<th>类型</th>
<th>说明</th>
</tr>
</thead>
<tbody>
<tr>
<td><code>code</code></td>
<td><code>String</code> 类型</td>
<td>包含有效 JavaScript 代码的字符串</td>
</tr>
</tbody>
</table>
<p>⚠️ <strong>注意</strong>： 参数<code>code</code>必须是原始字符串，不能是 String 对象形式。如果参数 <code>code</code> 不是原始字符串，则 <code>eval()</code> 函数不会执行代码，并且将其不作任何改变地返回。</p>
<p>如果参数 <code>code</code> 中的 JavaScript 代码不合法，将会引发异常。</p>
<p><code>eval()</code> 函数的返回值是任意类型，其返回值由参数 <code>code</code> 中具体的 JavaScript 代码决定。</p>
<h2>说明</h2>
<ul>
<li>传递给 <code>eval()</code> 函数的代码执行时所在的上下文和调用 <code>eval()</code> 函数时的上下文一样（也就是说，作用域不变）。</li>
<li>请自行确认 <code>code</code> 代码的来源是可信的，否则使用 <code>eval()</code> 函数存在一定的安全隐患。</li>
</ul>
<h2>示例</h2>
<pre class="hljs"><code class="language-js"><span class="hljs-keyword">let</span> x = <span class="hljs-number">2</span>,
  y = <span class="hljs-number">39</span>,
  z = <span class="hljs-string">&#x27;42&#x27;</span>;

<span class="hljs-built_in">eval</span>(<span class="hljs-string">&#x27;x + y + 1&#x27;</span>);
<span class="hljs-comment">// 42</span>

<span class="hljs-built_in">eval</span>(z);
<span class="hljs-comment">// 42</span>
</code></pre>
