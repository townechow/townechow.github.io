c:T115e,<h1>Array.prototype.some()</h1>
<p><code>Array.prototype.some()</code> 方法用于判定数组中是否存在一个成员符合判定函数判定条件。</p>
<h2>语法</h2>
<p>语法：</p>
<pre class="hljs"><code class="language-js">arr.<span class="hljs-title function_">some</span>( callback [, thisArg ] )
</code></pre>
<p>类型声明：</p>
<pre class="hljs"><code class="language-ts"><span class="hljs-keyword">interface</span> <span class="hljs-title class_">Array</span>&lt;T&gt; {
  <span class="hljs-title function_">some</span>(
    <span class="hljs-attr">predicate</span>: <span class="hljs-function">(<span class="hljs-params"><span class="hljs-attr">value</span>: T, <span class="hljs-attr">index</span>: <span class="hljs-built_in">number</span>, <span class="hljs-attr">array</span>: <span class="hljs-keyword">readonly</span> T[]</span>) =&gt;</span> <span class="hljs-built_in">unknown</span>,
    <span class="hljs-attr">thisArg</span>?: <span class="hljs-built_in">any</span>
  ): <span class="hljs-built_in">boolean</span>;
}
</code></pre>
<p>参数说明：</p>

<p><code>callback</code> 函数的参数：</p>
<ul>
<li><code>currentValue</code>：当前数组中处理的元素</li>
<li><code>index</code>：数组中正处理的当前元素的索引</li>
<li><code>array</code>：被调用的数组</li>
</ul>
<h2>方法说明</h2>
<ul>
<li>执行该方法会为数组每个成员执行一次回调函数，回调函数需要通过判断代码块后，返回布尔值作为该成员是否通过检测的凭证。当执行回调函数时遇到第一个判定为 <code>true</code> 的，则立即跳出迭代，返回 <code>true</code>，否则，全部数组成员都执行一次回调函数，没有数组成员通过判定，则返回 <code>false</code>。</li>
<li>回调函数只会为那些已经被赋值的索引调用，不会为那些被删除或从来没有被赋值的索引调用。</li>
<li>如果为实例方法提供一个 <code>thisArg</code> 参数，则该参数为调用回调函数时的 <code>this</code> 值。如果省略该参数，则为回调函数被调用时的 <code>this</code> 值，在非严格模式下为全局对象，在严格模式下传入 <code>undefined</code>。</li>
<li>遍历的数组成员范围在第一次调用回调函数之前就已确定了。在调用 <code>some()</code> 之后添加到数组中的成员不会被回调函数访问到。如果数组中存在的成员被更改，则他们传入回调函数的值是 <code>some()</code> 访问到他们那一刻的值。那些被删除的成员或未被赋值的成员将不会被访问到。</li>
</ul>
