c:T9c9,<h1>Proxy - handler.setPrototypeOf</h1>
<p><code>handler.setPrototypeOf()</code> 方法用于拦截 <code>Object.setPrototypeOf</code> 操作。</p>
<h2>语法</h2>
<pre class="hljs language-js"  style=--lang:"js" ><code><span class="hljs-keyword">const</span> proxy = <span class="hljs-keyword">new</span> <span class="hljs-title class_">Proxy</span>(target, {
  <span class="hljs-attr">setPrototypeOf</span>: <span class="hljs-keyword">function</span> (<span class="hljs-params">target, prototype</span>) {
    <span class="hljs-comment">// do something</span>
  },
});
</code></pre>

<h2>说明</h2>
<h3>拦截</h3>
<p>该方法会拦截目标对象的以下操作：</p>
<ul>
<li><code>Object.setPrototypeOf()</code></li>
<li><code>Reflect.setPrototypeOf()</code></li>
</ul>
<h3>约束</h3>
<p>如果违背了以下的约束，<code>proxy</code> 会抛出 TypeError：</p>
<ul>
<li>如果 <code>target</code> 不可扩展，原型参数必须与 <code>Object.getPrototypeOf(target)</code> 的值相同</li>
</ul>
<h2>示例</h2>
<p>以下代码演示了如何拦截 <code>Object.setPrototypeOf()</code>：</p>
<pre class="hljs language-js"  style=--lang:"js" ><code><span class="hljs-keyword">const</span> handler = {
  <span class="hljs-title function_">setPrototypeOf</span>(<span class="hljs-params">target, proto</span>) {
    <span class="hljs-keyword">throw</span> <span class="hljs-keyword">new</span> <span class="hljs-title class_">Error</span>(<span class="hljs-string">&#x27;Changing the prototype is forbidden&#x27;</span>);
  },
};

<span class="hljs-keyword">const</span> proto = {};

<span class="hljs-keyword">const</span> target = <span class="hljs-keyword">function</span> (<span class="hljs-params"></span>) {};

<span class="hljs-keyword">const</span> proxy = <span class="hljs-keyword">new</span> <span class="hljs-title class_">Proxy</span>(target, handler);

<span class="hljs-title class_">Object</span>.<span class="hljs-title function_">setPrototypeOf</span>(proxy, proto);
<span class="hljs-comment">// Error: Changing the prototype is forbidden</span>
</code></pre>
<p>注意，该方法只能返回布尔值，否则会被自动转为布尔值。另外，如果目标对象不可扩展（non-extensible），<code>setPrototypeOf()</code> 方法不得改变目标对象的原型。</p>
