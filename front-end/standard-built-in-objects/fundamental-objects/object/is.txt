c:Teb2,<h1>Object.is</h1>
<p><code>Object.is()</code> 方法用于判断两个值是否是相同的值。</p>
<h2>语法</h2>
<pre class="hljs"><code class="language-js"><span class="hljs-title class_">Object</span>.<span class="hljs-title function_">is</span>(value1, value2);
</code></pre>
<table>
<thead>
<tr>
<th>参数</th>
<th>说明</th>
<th>类型</th>
</tr>
</thead>
<tbody>
<tr>
<td>value1</td>
<td>比较值 1</td>
<td>any</td>
</tr>
<tr>
<td>value2</td>
<td>比较值 2</td>
<td>any</td>
</tr>
</tbody>
</table>
<p>返回判断表达式的结果。</p>
<h2>描述</h2>
<p>判断下列任何一项成立，则两个值相同：</p>
<ul>
<li>两个值均为 <code>undefined</code></li>
<li>两个值均为 <code>null</code></li>
<li>两个值都是 <code>true</code> 或 <code>false</code></li>
<li>两个值是由相同个数的字符按照相同顺序组成的字符串</li>
<li>两个值指向同一个对象</li>
<li>两个值都是数字并且
<ul>
<li>都是正零 <code>+0</code></li>
<li>都是负零 <code>-0</code></li>
<li>都是 <code>NaN</code></li>
<li>都是除零和 <code>NaN</code> 外的其他同一个数字</li>
</ul>
</li>
</ul>
<p>这种相等性判断逻辑和传统的 <code>==</code> 运算不同，<code>==</code> 运算符会对它两边的操作数做<strong>隐式类型转换</strong>，然后才进行相等性比较，（所以才会有类似 <code>&quot;&quot; == false</code> 等于 <code>true</code> 的现象），但 <code>Object.is</code> 不会做这种类型转换。</p>
<p>这与 <code>===</code> 运算符的判定方式也不一样。<code>===</code> 运算符（和 <code>==</code> 运算符）将数字值 <code>-0</code> 和 <code>+0</code> 视为相等，并认为 <code>Number.NaN</code> 不等于 <code>NaN</code>。</p>
