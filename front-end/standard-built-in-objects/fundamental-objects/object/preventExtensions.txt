c:T9dc,<h1>Object.preventExtensions</h1>
<p><code>Object.preventExtensions()</code> 方法用于令指定对象无法再添加新的属性。</p>
<h2>语法</h2>
<pre class="hljs"><code class="language-js"><span class="hljs-title class_">Object</span>.<span class="hljs-title function_">preventExtensions</span>(O);
</code></pre>
<table>
<thead>
<tr>
<th>参数</th>
<th>说明</th>
<th>类型</th>
</tr>
</thead>
<tbody>
<tr>
<td>O</td>
<td>将标记为不可扩展的对象</td>
<td>object</td>
</tr>
</tbody>
</table>
<p>返回处理后的对象。</p>
<h2>描述</h2>
<p>如果一个对象可以添加新的属性，则这个对象是可扩展的。</p>
<p><code>Object.preventExtensions</code> 能将对象标记为不可扩展，因此它将永远不会具有超出它被标记为不可扩展的 Properties。</p>
<p>⚠️ <strong>注意</strong>：一般来说，不可扩展对象的属性可能仍然可被删除。</p>
<p>该方法仅阻止添加自身的属性。但属性仍然可以添加到对象原型。</p>
<p>一旦使其不可扩展，就无法再对象进行扩展。</p>
<h2>示例</h2>
<p>字面量方式创建的对象默认是可扩展的。</p>
<pre class="hljs"><code class="language-js"><span class="hljs-keyword">const</span> foo = {};
<span class="hljs-title class_">Object</span>.<span class="hljs-title function_">isExtensible</span>(foo);
<span class="hljs-comment">// true</span>
</code></pre>
<p>但是可以改变。</p>
<pre class="hljs"><code class="language-js"><span class="hljs-title class_">Object</span>.<span class="hljs-title function_">preventExtensions</span>(foo);

<span class="hljs-title class_">Object</span>.<span class="hljs-title function_">isExtensible</span>(foo);
<span class="hljs-comment">// false</span>
</code></pre>
<p>使用 <code>Object.defineProperty</code> 方法为一个可扩展的对象添加新属性会抛出异常。</p>
<pre class="hljs"><code class="language-js"><span class="hljs-keyword">const</span> nonExtensible = { <span class="hljs-attr">removalbe</span>: <span class="hljs-literal">true</span> };

<span class="hljs-title class_">Object</span>.<span class="hljs-title function_">preventExtensions</span>(nonExtensible);

<span class="hljs-title class_">Object</span>.<span class="hljs-title function_">defineProperty</span>(nonExtensible, <span class="hljs-string">&#x27;new&#x27;</span>, { <span class="hljs-attr">value</span>: <span class="hljs-number">5678</span> });
<span class="hljs-comment">// throw TypeError Exception</span>
</code></pre>
