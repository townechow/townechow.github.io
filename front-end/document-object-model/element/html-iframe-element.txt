<h3>沙箱模式</h3>
<p><code>iframe</code> 的沙箱模式可以提供一些额外的配置，当你把 <code>iframe</code> 置为沙箱的时候，意味着沙箱内的内容的行为 <strong>全凭你控制</strong> 了。</p>
<p>例如在 <code>iframe</code> 中设置：</p>
<pre class="hljs"><code class="language-html"><span class="hljs-tag">&lt;<span class="hljs-name">iframe</span> <span class="hljs-attr">sandbox</span> <span class="hljs-attr">src</span>=<span class="hljs-string">&quot;http://127.0.0.1:3000/iframe1.html&quot;</span>&gt;</span><span class="hljs-tag">&lt;/<span class="hljs-name">iframe</span>&gt;</span>
</code></pre>
<p>框架文件将会被完全沙箱化，并有以下限制：</p>
<ul>
<li>JavaScript 将不会在框架文档中执行。这不仅包括通过 <code>script</code> 标签加载的 JavaScript 脚本文件，还包括内联事件处理程序和 <code>javascript: URLs</code>。这还意味着 <code>noscript</code> 标签中的内容会被显示，就像用户自己禁用了脚本一样</li>
<li>框架文档被加载到唯一的源，这意味着所有同源检查都将失败；唯一的源不匹配其他源，甚至它们自己。在其他影响中，这意味着文档不能访问其他源的 <code>cookie</code> 或任何其他存储机制（ DOM 存储、索引数据库等）中的数据</li>
<li>框架文档不能创建新窗口或对话框（例如，通过 <code>window.open</code> 或 <code>target=&quot;blank&quot;</code>）</li>
<li>表单不能提交</li>
<li>插件不会被加载</li>
<li>框架文档只能导航自己，而不能导航其顶级父文档。设置 <code>window.top.location</code> 将抛出一个异常，点击带有 <code>target=&quot;_top&quot;</code> 的链接将不起作用</li>
<li>自动触发的功能（自动聚焦的表单元素、自动播放视频等）将被阻止</li>
<li>无法获得 Pointer Lock</li>
<li>在框架文档包含的 <code>iframes</code> 上忽略 <code>seamless</code> 属性</li>
</ul>
<p>文件被加载到一个完全沙箱化的 <code>iframe</code> 中，带来的风险非常小，但这是十分苛刻的。当然，这样做也没有太大的价值：对于一些静态内容，可以使用一个完整的沙箱，但大多数情况下，可以放宽松一些。</p>
<p>通过设置以下值控制：</p>
<ul>
<li><code>allow-same-orign</code>：允许将内容作为普通来源对待，否则会被设为来自一个独立的源</li>
<li><code>allow-top-navigation</code>：允许包含文档导航内容</li>
<li><code>allow-forms</code>：允许表单提交</li>
<li><code>allow-scripts</code>：允许脚本执行</li>
<li><code>allow-modals</code>：允许模态窗口</li>
<li><code>allow-orientation-lock</code>：允许锁定父窗口屏幕方向</li>
<li><code>allow-pointer-lock</code>：允许使用指针锁 API</li>
<li><code>allow-popups</code>：允许弹出窗口</li>
<li><code>allow-popups-to-escape-sandbox</code>：允许弹出沙箱窗口</li>
<li><code>allow-presentation</code>：允许控制 Session</li>
<li><code>”“</code>：允许上述所有规则，默认</li>
</ul>
<h2>方法</h2>
<h3>导航方法</h3>
<ul>
<li>reload 重新加载</li>
<li>stop 停止加载</li>
<li>getCanGoBack 是否可后退</li>
<li>goBack 后退</li>
<li>getCanGoForward 是否可前进</li>
<li>goForward 前进</li>
</ul>
<h3>管理方法</h3>
<ul>
<li>executeScript 指定加载完成后执行的脚本</li>
<li>purgeHistroy 清理与浏览器有关的资源</li>
<li>setVisible 修改浏览器 <code>iframe</code> 的可见性，这会影响资源分配和函数的资源占用率</li>
<li>getVisible 指示当前浏览器 <code>iframe</code> 的可见性</li>
<li>setActive 设置当前 <code>iframe</code> 为活动的 <code>frame</code>，对进程管理器如何划分优先级有影响</li>
<li>getActive 指示当前浏览器 <code>iframe</code> 是否为当前活动的 <code>frame</code>
setInputMethodActive 设置当前浏览器 <code>iframe</code> 是活动的输入法编辑器窗口而其他不适
setNfcFocus</li>
</ul>
<h3>音频相关方法</h3>
<ul>
<li>getVolume 当前音量</li>
<li>setVolume 设置音量</li>
<li>mute 播放所有音频静音</li>
<li>unmute 取消播放发所有音频的静音</li>
<li>getMuted 指示内嵌网页是否被静音</li>
</ul>
<h3>搜索方法</h3>
<ul>
<li>findAll</li>
<li>findNext</li>
<li>clearMatch</li>
</ul>
<h3>事件相关方法</h3>
<ul>
<li>sendMouseEvent</li>
<li>sendTouchEvent</li>
<li>addNextPaintListener</li>
<li>removeNextPainListener</li>
</ul>

<ul>
