<h2>orientationchange 事件</h2>
<p>苹果公司为移动 Safari 中添加了 <code>orientationchange</code> 事件，以便开发人员能够确定用户何时将设备由横向查看模式切换为纵向查看模式。移动 Safari 的 <code>window.orientation</code> 属性中可能包含 3 个值：</p>
<ul>
<li>0：表示肖像模式</li>
<li>90：表示向左旋转的横向模式（“主屏幕”按钮在右侧）</li>
<li>-90：表示向右旋转的横向模式（“主屏幕”按钮在左侧）</li>
</ul>
<p>相关文档中还提到一个值，即 180 表示 iPhone 头朝下；但这种模式至今尚未得到支持。</p>
<p>只要用户改变了设备的查看模式，就会触发 <code>orientationchange</code> 事件。此时的 <code>event</code> 对象不包含任何有价值的信息，因为唯一相关的信息可以通过 <code>window.orientation</code> 访问到。下面是使用这个事件的典型示例。</p>
<pre class="hljs language-js"  style=--lang:"js" ><code><span class="hljs-title class_">EventUtil</span>.<span class="hljs-title function_">addHandler</span>(<span class="hljs-variable language_">window</span>, <span class="hljs-string">&#x27;load&#x27;</span>, <span class="hljs-keyword">function</span>(<span class="hljs-params">event</span>) {
  <span class="hljs-keyword">const</span> div = <span class="hljs-variable language_">document</span>.<span class="hljs-title function_">getElementById</span>(<span class="hljs-string">&#x27;myDiv&#x27;</span>);

  });
});
</code></pre>
<p>在这个例子中，当触发 <code>load</code> 事件时会显示最初的方向信息。然后，添加了处理 <code>orientationchange</code> 事件的处理程序。只要发生这个事件，就会有表示新方向的信息更新页面中的消息。</p>
<p>所有 iOS 设备都支持 <code>orientationchange</code> 事件和 <code>window.orientation</code> 属性。</p>
<h2>MozOrientation 事件</h2>
<p>Firefox3.6 为检查设备的方向引入了一个名为 <code>MozOrientation</code> 的新事件。（前缀 Moz 表示这是特定于浏览器开发商的事件，不是标准事件。）当设备的加速计检测到设备方向改变时，就会触发这个事件。但这个事件与 iOS 中的 <code>orientationchange</code> 事件不同，该事件只能提供一个平面的方向变化。由于 <code>MozOrientation</code> 事件是在 <code>window</code> 对象上触发的，所以可以使用以下代码来处理。</p>
