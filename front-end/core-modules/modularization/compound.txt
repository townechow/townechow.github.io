c:T1117,<h1>模块导入/导出的复合写法</h1>
<p>如果在一个模块之中，先输入后输出同一个模块，<code>import</code> 语句可以与 <code>export</code> 语句写在一起。</p>
<h2>模块整体转发</h2>
<p>从 <code>module</code> 模块整体导入后，直接完整导出。</p>
<pre class="hljs"><code class="language-js"><span class="hljs-keyword">export</span> * <span class="hljs-keyword">from</span> <span class="hljs-string">&#x27;module&#x27;</span>
</code></pre>
<h2>模块部分接口转发</h2>
<p>从 <code>module</code> 模块导入 <code>foo</code> 和 <code>bar</code>，并直接导出这两个接口。</p>
<pre class="hljs"><code class="language-js"><span class="hljs-keyword">export</span> { foo, bar } <span class="hljs-keyword">from</span> <span class="hljs-string">&#x27;module&#x27;</span>

<span class="hljs-comment">// 可以理解为</span>
<span class="hljs-keyword">import</span> { foo, bar } <span class="hljs-keyword">from</span> <span class="hljs-string">&#x27;module&#x27;</span>
<span class="hljs-keyword">export</span> { foo, bar }
</code></pre>
<p>实际上，这样的写法只是相当于对外转发了这两个接口，当前模块不能直接使用这两个接口。</p>
<h2>模块部分重命名转发</h2>
<p>模块导入的接口重命名，从 <code>module</code> 导入 <code>foo</code> 接口，并以 <code>newFoo</code> 的名义导出。</p>
<pre class="hljs"><code class="language-js"><span class="hljs-keyword">export</span> { foo <span class="hljs-keyword">as</span> newFoo } <span class="hljs-keyword">from</span> <span class="hljs-string">&#x27;module&#x27;</span>
</code></pre>
<h2>默认模块转发</h2>
<pre class="hljs"><code class="language-js"><span class="hljs-keyword">export</span> { <span class="hljs-keyword">default</span> } <span class="hljs-keyword">from</span> <span class="hljs-string">&#x27;module&#x27;</span>
</code></pre>
<h2>命名模块改默认模块</h2>
<pre class="hljs"><code class="language-js"><span class="hljs-keyword">export</span> { foo <span class="hljs-keyword">as</span> <span class="hljs-keyword">default</span> } <span class="hljs-keyword">from</span> <span class="hljs-string">&#x27;./module&#x27;</span>

<span class="hljs-comment">// 等同于</span>
<span class="hljs-keyword">import</span> { foo } <span class="hljs-keyword">from</span> <span class="hljs-string">&#x27;./module&#x27;</span>
<span class="hljs-keyword">export</span> <span class="hljs-keyword">default</span> foo
</code></pre>
<h2>默认模块改命名模块</h2>
<pre class="hljs"><code class="language-js"><span class="hljs-keyword">export</span> { <span class="hljs-keyword">default</span> <span class="hljs-keyword">as</span> foo } <span class="hljs-keyword">from</span> <span class="hljs-string">&#x27;./module&#x27;</span>
</code></pre>
<h2>无对应写法场景</h2>
<p>下面三种写法，没有对应的复合写法：</p>
<ul>
<li>命名模块重命名转发</li>
<li>默认模块转发</li>
<li>整体和部分模块共同转发</li>
</ul>
<pre class="hljs"><code class="language-js"><span class="hljs-comment">// 命名模块重命名转发</span>
<span class="hljs-keyword">import</span> * <span class="hljs-keyword">as</span> foo <span class="hljs-keyword">from</span> <span class="hljs-string">&#x27;./module&#x27;</span>

<span class="hljs-comment">// 默认模块转发</span>
<span class="hljs-keyword">import</span> foo <span class="hljs-keyword">from</span> <span class="hljs-string">&#x27;./module&#x27;</span>

<span class="hljs-comment">// 整体和部分模块共同转发</span>
<span class="hljs-keyword">import</span> foo , { namedFoo } <span class="hljs-keyword">from</span> <span class="hljs-string">&#x27;./module&#x27;</span>
</code></pre>
<p>为了做到形式的对称，现在有提案，提出补上这三种复合写法。</p>
<pre class="hljs"><code class="language-js"><span class="hljs-keyword">export</span> * <span class="hljs-keyword">as</span> foo <span class="hljs-keyword">from</span> <span class="hljs-string">&#x27;./module&#x27;</span>

<span class="hljs-keyword">export</span> foo <span class="hljs-keyword">from</span> <span class="hljs-string">&#x27;./module&#x27;</span>

<span class="hljs-keyword">export</span> foo , { namedFoo } <span class="hljs-keyword">from</span> <span class="hljs-string">&#x27;./module&#x27;</span>
</code></pre>
