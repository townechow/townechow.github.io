c:Tfae,<h1>单例模式</h1>
<p><strong>单例模式（Singleton）</strong> 指保证一个类仅有一个实例，并提供一个访问它的全局访问点。</p>
<h2>功能</h2>
<ul>
<li>单例模式能保证全局的唯一性，可以减少命名变量</li>
<li>单例模式在一定情况下可以节约内存，减少过多的类生成需要的内存和运行时间</li>
<li>把代码都放在一个类里面维护，实现了高内聚</li>
</ul>
<p>优点：</p>
<ol>
<li>提供了对唯一实例的受控访问</li>
<li>避免对共享资源的多重占用</li>
<li>节约系统资源</li>
</ol>
<p>缺点：</p>
<ol>
<li>扩展性差</li>
<li>职责过重</li>
</ol>
<h2>代码示例</h2>
<p>全局作用域中的全局变量不是单例模式，全局变量会存在很大的隐患，随着项目的体积和功能日益增大，很容易出现命名冲突、作用域内变量污染和变量覆盖等问题，给开发人员带来很多苦恼。</p>
<p>所以要减少全局变量使用，即使用全局变量也要将污染降到最低。</p>
<h3>命名空间</h3>
<p>命名空间可以减少全局变量的数量，可以使用对象字面量将这一类的变量作为它的属性进行访问。</p>
<pre class="hljs"><code class="language-js"><span class="hljs-keyword">var</span> <span class="hljs-title class_">Singleton</span> = {

<h3>使用闭包封装私有变量</h3>
<p>使用 IIFI 立即执行函数表达式，让 JavaScript 编译器不在认为这是一个函数声明，而是 <strong>立即执行函数</strong>，将私有变量保存在它的闭包环境中，暴露可以访问私有变量的接口。类似创建一个块级作用域，和其他作用域的变量隔离。</p>
<pre class="hljs"><code class="language-js"><span class="hljs-keyword">var</span> <span class="hljs-title class_">Singleton</span> = (<span class="hljs-keyword">function</span> (<span class="hljs-params"></span>) {
