c:T135e,<h1>File API</h1>
<p>File 对象接口提供有关文件的信息，并允许网页中的 JavaScript 访问起内容。</p>
<p>通常情况下，File 对象是来自：</p>
<ul>
<li>用户在一个 <code>&lt;input type=&quot;file&quot;&gt;</code> 元素上选择文件后返回的 <a href="/front-end/browser-object-model/binary-data-and-files/file-list">FileList</a> 对象</li>
<li>拖拽中生成的 DataTransfer 对象</li>
<li>HTMLCanvasElement 上的 <code>mozGetAsFile()</code> API</li>
</ul>
<p>在 Gecko 中，特权代码可以创建代表任何本地文件的 File 对象，而无需用户交互。</p>
<p>File 对象是特殊类型的 Blob 对象，且可以用在任意的 Blob 类型的 <code>context</code> 中。比如说，<a href="/front-end/browser-object-model/binary-data-and-files/file-reader">FileReader</a>，<code>URL.createObjectURL()</code>，<code>createImageBitmap()</code> 及 <code>XMLHttpRequest.send()</code> 都能处理 Blob 和 File。</p>
<h2>基本用法</h2>
<p>通过 <code>new</code> 操作符构建文件对象 File。</p>
<pre class="hljs"><code class="language-js"><span class="hljs-keyword">const</span> file = <span class="hljs-keyword">new</span> <span class="hljs-title class_">File</span>();
</code></pre>
<h2>属性和方法</h2>
<p>File 对象也继承了 Blob 对象的属性。以下属性均为 <strong>只读</strong> 属性，不可修改。</p>
<table>
<thead>
<tr>
<th style="text-align:left">属性</th>
<th style="text-align:left">说明</th>
</tr>
</thead>
<tbody>
<tr>
<td style="text-align:left"><code>File.lastModified</code></td>
<td style="text-align:left">当前引用文件 <strong>最后的修改时间</strong>，为自 1970 年 1 月 1 日 0 时 0 分以来的毫秒数</td>
</tr>
<tr>
<td style="text-align:left"><code>File.lastModifiedDate</code> 🗑</td>
<td style="text-align:left">当前引用文件 <strong>最后的修改时间</strong>，为 Date 对象</td>
</tr>
<tr>
<td style="text-align:left"><code>File.name</code></td>
<td style="text-align:left">当前引用文件的 <strong>名称</strong></td>
</tr>
<tr>
<td style="text-align:left"><code>File.size</code></td>
<td style="text-align:left">当前引用文件的 <strong>大小</strong></td>
</tr>
<tr>
<td style="text-align:left"><code>File.webkitRelativePath</code> ⚠️</td>
<td style="text-align:left">当前引用文件的 <strong>路径或 URL</strong></td>
</tr>
<tr>
<td style="text-align:left"><code>File.type</code></td>
<td style="text-align:left">当前引用文件的多用途互联网邮件 <strong>扩展类型</strong></td>
</tr>
</tbody>
</table>
<p>File 对象接口没有定义任何方法，但是继承了 Blob 对象接口的方法。</p>
<h2>注意事项</h2>
<ul>
<li>下面的非标准的属性及方法在 Gecko 7（Firefox 7.0 / Thunderbird 7.0 / SeaMonkey 2.4)）里就被移除：
<ul>
<li><code>File.fileName</code></li>
<li><code>File.fileSize</code></li>
<li><code>File.getAsBinary()</code></li>
<li><code>File.getAsDataURL()</code></li>
<li><code>File.getAsText</code></li>
</ul>
</li>
</ul>
<p>应该使用 <code>File.name</code> ，<code>Blob.size</code>，和 <code>FileReader</code> 的方法来代替。</p>
