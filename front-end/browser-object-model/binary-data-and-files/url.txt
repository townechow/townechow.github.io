c:Tffb,<h1>URL API</h1>
<blockquote>
<p>⚠️ 这是一个实验中的功能，此功能某些浏览器尚在开发中，请参考浏览器兼容性表格以得到在不同浏览器中适合使用的前缀。由于该功能对应的标准文档可能被重新修订，所以在未来版本的浏览器中该功能的语法和行为可能随之改变。</p>
</blockquote>
<p>URL 接口是一个包含若干静态方法的对象，用来创建 URLs。</p>
<p>当使用一个没有实现该构造器的用户代理时，可以通过 Window.URL 属性来访问该对象（基于 Webkit 和 Blink 内核的浏览器均可用 <code>Window.webkitURL</code> 代替）。</p>
<h2>基本用法</h2>
<pre class="hljs"><code class="language-js"><span class="hljs-keyword">const</span> url = <span class="hljs-keyword">new</span> <span class="hljs-title function_">URL</span>();
</code></pre>
<h2>属性</h2>
<p>实现 URLUtils 中定义的属性。</p>
<table>
<thead>
<tr>
<th style="text-align:left">属性</th>
<th style="text-align:left">说明</th>
</tr>
</thead>
<tbody>
<tr>
<td style="text-align:left">URLUtils.href</td>
<td style="text-align:left">包含完整 URL 的 DOMString。</td>
</tr>
<tr>
<td style="text-align:left">URLUtils.protocol</td>
<td style="text-align:left">包含 URL 协议名的 DOMString。</td>
</tr>
<tr>
<td style="text-align:left">URLUtils.host</td>
<td style="text-align:left">包含 URL 域名，<code>:</code> 和端口号的 DOMString。</td>

<tr>
<td style="text-align:left">URLUtils.hostname</td>
<td style="text-align:left">包含 URL 域名的 DOMString。</td>

<tr>

<td style="text-align:left">包含 URL 端口号的 DOMString。</td>
</tr>
<tr>
<td style="text-align:left">URLUtils.pathname</td>
<td style="text-align:left">以 <code>/</code> 起头紧跟着 URL 文件路径的 DOMString。</td>
</tr>
<tr>
<td style="text-align:left">URLUtils.search</td>
<td style="text-align:left">以 <code>?</code> 起头紧跟着 URL 请求参数的 DOMString。</td>

<tr>
<td style="text-align:left">URLUtils.hash</td>

</tr>
<tr>
<td style="text-align:left">URLUtils.username</td>
<td style="text-align:left">包含在域名前面指定的用户名 DOMString。</td>
</tr>
<tr>
<td style="text-align:left">URLUtils.password</td>
<td style="text-align:left">包含在域名前面指定的密码的 DOMString。</td>
</tr>
<tr>
<td style="text-align:left">URLUtils.origin</td>
<td style="text-align:left">返回一个包含协议名、域名和端口号的 DOMString。</td>
</tr>

<td style="text-align:left">URLUtils.searchParams</td>
<td style="text-align:left">返回一个用来访问当前 URL GET 请求参数的 URLSearchParams 对象。</td>
